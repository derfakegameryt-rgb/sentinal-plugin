# Chest-UI Theme (Dark + Blue Accent) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restyle every chest GUI to a dark + blue-accent theme centrally (black filler, blue control bar + menu borders) with no per-GUI edits and no loss of content slots.

**Architecture:** Change `Items.filler()` to black glass and add `Items.accent()` (light-blue glass); point `Gui.border()` and the `Gui.navBar()` control-row fill at the accent. Every GUI restyles automatically.

**Tech Stack:** Java 21, Paper API, JUnit 5, MockBukkit.

## Global Constraints

- `Items.filler()` → `BLACK_STAINED_GLASS_PANE`; `Items.accent()` → `LIGHT_BLUE_STAINED_GLASS_PANE` (closest stock glass to the plugin blue `#3B82F6`). Both keep a single blank (space) display name.
- Borders (menus) and the bottom control row (45–53) use the accent; empty CONTENT slots use the black filler. No content slots lost; no per-GUI layout edits.
- Existing GUI tests stay green except the one border-colour assertion, which is updated.
- Do NOT `git add -A` (gitignored `.claude/` + `.superpowers/`). Stage explicit paths. Run `./gradlew test` before commit.
