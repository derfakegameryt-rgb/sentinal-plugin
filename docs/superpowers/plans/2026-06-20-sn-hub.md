# `/sn` Hub Menu Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `/sn` (no argument) open the `AdminPanelGui` hub — which gains Player Manager, Vanish, and Staff Chat entries — and remove Vanish/Staff Chat from the player list.

**Architecture:** Repurpose the existing `AdminPanelGui` (27 slots) as the `/sn` hub: drop its now-pointless Back button (freeing interior slots) and add Player Manager + Vanish + Staff Chat entries. Point `SentinelCommand`'s no-arg branch at it. Strip Vanish/Staff Chat from `PlayersGui` and relabel its existing "Admin Panel" button to "Back to menu".

**Tech Stack:** Java 21, Paper API, MockBukkit, JUnit 5.

## Global Constraints

- No behavior change to the underlying actions: `plugin.vanish().toggle(Player)` and `plugin.staffChat().toggle(UUID)` are moved, not modified; their existing message keys (`vanish-on/off`, `staffchat-on/off`) are reused.
- `/sn <player>` and `/sn <subcommand>` are unchanged; only the no-arg branch changes.
- `AdminPanelGui` stays 27 slots; all entries use explicit slot constants within the bordered interior (slots 10–16 and 19–25); reserve two free interior slots (23, 24) for the future Audit/Stats entries.
- Full existing test suite stays green. Run `./gradlew test` before committing.
