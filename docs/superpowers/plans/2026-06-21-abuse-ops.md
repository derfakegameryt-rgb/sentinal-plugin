# Abuse Protection & Operations Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a reusable cooldown for `/report` + `/appeal`, a curated `debug` logging switch, and a permission audit (gap-fix + docs) — without changing healthy behaviour.

**Architecture:** A `CooldownManager` util backs per-player rate limits on the abuse-prone commands (staff bypass). `Sentinel.debug(msg)` is a no-op-unless-enabled logger used at a curated set of diagnostic points. A permission audit closes gaps using the existing `StaffPermissions` API and documents the model.

**Tech Stack:** Java 21, Paper API, JUnit 5, MockBukkit.

## Global Constraints

- Cooldowns apply ONLY to `/report` and `/appeal`. Holders of `sentinel.use` bypass them. `0` seconds disables a cooldown. Config read live (so `/sentinel reload` applies).
- `debug` is a curated set of diagnostic logs — NO logging in hot per-tick / per-message / per-DB-call paths. `debug(msg)` never throws and is a no-op when off.
- Permission audit keeps the CURRENT model (no new node granularity): operator commands gate on `sentinel.use` or the existing per-type node; punishment actions use `canPerform(type)`; GUI actions mirror command checks. Reuse the `no-permission` message.
- No new dependencies. Existing suite stays green.
- Do NOT `git add -A` (gitignored `.claude/` + `.superpowers/`). Stage explicit paths. Run `./gradlew test` before each commit.

```java
package de.derfakegamer.sentinel.util;

import java.util.UUID;
import java.util.concurrent.ConcurrentHashMap;

/** In-memory per-(player, action) cooldowns. Server-local anti-spam; never throws. */
public final class CooldownManager {
    private final ConcurrentHashMap<String, Long> last = new ConcurrentHashMap<>();
    private static String k(UUID id, String key) { return id + ":" + key; }

    /** Records {@code now} and allows when off cooldown (or {@code cooldownMillis <= 0}); else leaves state and returns false. */
    public boolean tryUse(UUID id, String key, long cooldownMillis, long now) {
        if (cooldownMillis <= 0) return true;
        String kk = k(id, key);
        Long prev = last.get(kk);
        if (prev != null && now - prev < cooldownMillis) return false;
        last.put(kk, now);
        return true;
    }

    public long remainingMillis(UUID id, String key, long cooldownMillis, long now) {
        if (cooldownMillis <= 0) return 0;
        Long prev = last.get(k(id, key));
        if (prev == null) return 0;
        return Math.max(0, cooldownMillis - (now - prev));
    }
}
```
